Introduction

The following section was built for class COMP200 (Introduction to Computing and Information Systems)

The task was of this project was to design a normalized database that could be used to collect and store information for a video rental outlet. The database is structured to support the following tasks:

  • Tracking rentals by title and by customers
  • Producing reports, such as rentals due on a certain date
  • Categorizng rentals by media type (e.g., Blu-Ray, DVD, Game)

Although the assigment did not require to produce examples of the reports listed, but only that the database be correctly structured to allow the query and reporting features of a DBMS.

Requirements

4 database tables populated with records

  • Customers - At least 10
  • Categories - 3 records, one describing each of the media types: DVD, Blu-Ray, or Game
  • Media - At least 30 records, representing a selection of DVD movies, Blu-Ray movies, and Games
  • Rentals - At least 20

I implemented CRUD functionality to the (Working Database) section which also covers searching functionality.
